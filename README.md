# Руководство пользователя GitHub для MCTs

Облачные службы, такие как Microsoft Azure, часто обновляются, что приводит к проблемам для сертифицированных тренеров Майкрософт (MCTS), так как они учат курсы и лабораторные шаги больше не соответствуют нашим облачным службам. . Из-за частоты изменений и того факта, что при изменении изменений может не быть никаких уведомлений, команде разработчиков курсов может быть трудно быстро определить и настроить изменения лаборатории.

Для решения этих проблем мы используем GitHub для публикации шагов лаборатории и скриптов лаборатории для курсов, охватывающих облачные службы, такие как Azure. Использование GitHub позволяет авторам курса и MCTs поддерживать текущее содержимое лаборатории с изменениями облачной службы. Использование GitHub позволяет MCTs предоставлять отзывы и предложения по изменениям лаборатории, а затем авторы курса могут быстро обновлять шаги лаборатории и скрипты.

При подготовке к обучению этим курсам следует убедиться, что вы используете последние шаги лаборатории и скрипты, скачав соответствующие файлы из GitHub.

Это руководство пользователя предназначено для MCTs, которые являются новыми для GitHub. Он содержит инструкции по подключению к GitHub, скачиванию и печати материалов курса, обновлению скриптов, используемых учащимися в лабораториях, и объясняя, как помочь убедиться, что содержимое курса остается текущим.

> **Примечание**. Поддержка microsoft Обучение для доступа к файлам на GitHub и поддержка навигации сайта GitHub ограничена только mcts, которые обучают этот курс.

GitHub не следует использовать для обсуждения технического содержимого в курсе или подготовки к курсу или его лабораторий. Это специально для решения изменений в лабораториях.

 
> **Примечание**. Чтобы обратиться к общим комментариям о курсе и демонстрации, или как подготовиться к курсу, используйте форумы MCT.

## Терминология GitHub

GitHub вводит терминологию, которая может быть новой для вас. В следующем списке содержатся термины и понятия, используемые в этом документе. Однако полный список терминов GitHub см. в [глоссарии](https://docs.github.com/en/get-started/quickstart/github-glossary) GitHub.

| Термин| Описание |
| - | - |
| Git и GitHub| Git — это программа с открытым кодом, отслеживание изменений, а GitHub — это сайт или решение, созданное на основе Git. Существуют другие веб-сайты и решения, которые используют Git в качестве серверной части. Вы будете использовать GitHub в первую очередь для проектов разработки с открытым кодом (общедоступного) и бесплатно для этих проектов. Однако если вы хотите использовать GitHub для проектов, которые являются частными, а не открытый код, необходимо зарегистрироваться для платной версии. |
| Репозиторий или репозиторий| Каждый проект в GitHub находится в репозитории или репозитории. Репозиторий содержит все файлы для проекта, включая документацию, и поддерживает журнал редакций. Репозиторий может быть общедоступным или частным. Вы можете использовать локальную копию репозитория на жестком диске компьютера или использовать репозиторий в GitHub. |
| Markdown| Это текстовый формат файла, который можно использовать для создания документации. Это текстовый и очень простой для обновления, что упрощает использование во время совместной работы. Затем GitHub отображает его как HTML. |
| GitHub с ароматизированным markdown (GFM)| Существует множество вариантов или вариантов формата файла Markdown. Версия GitHub, обычно называемая GFM, является одним из наиболее распространенных вариантов Markdown. Дополнительные сведения о GFM и о том, как использовать формат разметки для документов GFM, см. в статье "Приступая к написанию и форматированию на GitHub".https://help.github.com/articles/getting-started-with-writing-and-formatting-on-github/ |
| Вилка| Это копия другого репозитория, который находится в учетной записи GitHub, по сравнению с ветвью, которая находится в исходном репозитории. См. раздел "Ветвь" непосредственно ниже. |
| Ветвь| Это копия репозитория, который находится в том же репозитории, что и исходный. Вы можете объединить ветвь с исходным. |
| Получить| Это процесс получения копии последних изменений из интернет-репозитория. Однако выборка не объединяет изменения. |
| Извлечь| Это процесс получения последних изменений из онлайн-репозитория и объединения их с локальными изменениями. |
| Слияние| Это процесс получения изменений из одной ветви и их применения к другой. Это включает получение изменений из веб-репозитория, а затем применение их к локальной версии репозитория. |
| Запрос на вытягивание| Это набор предлагаемых изменений в репозитории, который пользователь отправляет, а владельцы или сотрудники репозитория могут принять или отклонить запрос на вытягивание. |
| Явно (толкать)| Это процесс отправки или отправки локальных изменений в репозиторий в Интернете. |
| Collaborator| Это пользователь GitHub, имеющий разрешения на добавление, удаление или изменение содержимого репозитория. |

## Получение уведомлений об обновлении, предложение изменений и совместная работа над проектами

Вы можете настроить интерфейс GitHub, чтобы получать уведомления при обновлении репозитория GitHub. Существует несколько способов регистрации для уведомлений, и многие из них связаны с множеством способов совместной работы над проектом. Для получения уведомлений можно выполнить следующие действия.

| Действие| Description |
| - | - |
| [Просмотр репозиториев](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/watching/)| При просмотре репозитория GitHub автоматически подписывает вас на уведомления о новых запросах на вытягивание или проблемы, созданные для этого конкретного репозитория. Вы автоматически просматриваете все создаваемые репозитории или для которых вы являетесь сотрудником. |
| [Запрос на вытягивание](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/pullrequest/)| При создании запроса на вытягивание и предложение владельцам репозитория принять изменения, вы автоматически подписываетесь на получение уведомлений о связанном обсуждении запроса на вытягивание. Чтобы создать запрос на вытягивание, необходимо сначала создать ветвь. |
| [Комментарии](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/comment/)| Когда вы делаете комментарии о запросе на вытягивание другого человека, GitHub подписывает вас автоматически на форум, который относится к этому комментариев, или вы можете вручную подписаться на форум. |
| [Проблемы](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/issue/)| Проблема — это предложение, вопрос или запрос, относящийся к репозиторию. Каждая проблема имеет свой собственный обсуждение, и вы можете подписаться на проблемы, или GitHub автоматически подписывает вас на создаваемые проблемы. |
| [Упоминает](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/mention/)| Когда другой пользователь упоминание вас в беседе, используя имя пользователя GitHub ([@username](https://github.com/username)), GitHub автоматически подписывается на обсуждение. |

> **Примечание.** Вы можете изменить способ и время получения уведомлений, а также отменить подписку на любые или все обсуждения.

## Отправка проблем или предложение изменений в инструкции лаборатории

Если у вас есть предложение или возникла ошибка в лаборатории, вы можете отправить запрос на вытягивание и запросить проблему. Если вы уже знаете исправление ошибки, рекомендуется отправить запрос на вытягивание; в противном случае отправьте проблему.

| Действие| Description |
| - | - |
| [Запрос на вытягивание](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/pullrequest/)| При создании запроса на вытягивание и предложения владельцев репозитория принять изменения, которые вы делаете, вы автоматически подписываетесь на получение уведомлений о связанном обсуждении запроса на вытягивание. Чтобы создать запрос на вытягивание, необходимо сначала создать ветвь. |
| [Комментарии](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/comment/)| Когда вы делаете комментарии о запросе на вытягивание другого человека, GitHub автоматически подписывает вас на форум, который относится к этому комментариев, или вы можете вручную подписаться на форум. |
| [Проблемы](https://microsoftlearning.github.io/MCT-User-Guide/collaboration/issue/)| Проблема — это предложение, вопрос или запрос, относящийся к репозиторию. Каждый вопрос имеет свое собственное обсуждение. Вы можете подписаться на проблемы или GitHub автоматически подписывает вас на создаваемые проблемы. |
